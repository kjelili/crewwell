# Documentation Sources Used

This implementation used the following documentation references and applied them directly in setup and delivery.

## Product source document

- `CrewWell.docx` (local project document)
  - Used to derive MVP scope and v2 roadmap:
    - check-in workflow
    - anonymous or named submissions
    - dashboard by crew/site
    - trend alerts
    - manager recommendations

## Build tooling documentation

- [Vite Getting Started](https://vite.dev/guide/)
  - Used for project structure, scripts, and production build process.

## Demo-video documentation

- [Remotion CLI render docs](https://remotion.dev/docs/cli/render)
  - Used to configure `video:render` command.
  - Windows note considered for CLI JSON props behavior.

- [Remotion docs](https://remotion.dev/docs)
  - Used for composition setup (`registerRoot`, `Composition`) and video pipeline approach.
